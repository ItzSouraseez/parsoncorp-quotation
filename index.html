CRM & integrations — scope detail
CRM Core
Built-in CRM for Parsoncorp
A dedicated CRM layer inside the admin panel — full visibility into every corporate client, their order history, deals, and communication timeline.
  • Per-client profile with company, GSTIN, contacts
  • Deal & order pipeline (Kanban + list view)
  • Activity log — calls, emails, meetings, notes
  • Assigned account manager per client
  • Client-level custom pricing & credit terms
  • Tags, segments & global client search
Per-Client Config
Per-client integration control panel
Each client (Google, Nvidia, Accenture) gets their own integration settings — independently toggle Salesforce and Tally, configure credentials, and map fields.
  • Per-client Salesforce / Tally toggle
  • Encrypted credentials vault (AES-256)
  • Field mapping configuration UI
  • Sync status & error log per client
  • Manual trigger + auto-sync schedule
  • Full audit trail of all sync events
Salesforce
Salesforce integration
Each client connects their own SF org via OAuth. Orders, contacts, and deals sync bi-directionally — automatically keeping both systems in sync.
  • Per-client OAuth 2.0 Salesforce login
  • Contact & account sync (SF ↔ CRM)
  • Order push to SF Opportunities
  • Deal stage mapping per client org
  • Webhook listener for SF-side updates
  • Configurable field mapping per org
Tally
Tally Prime integration
For clients using Tally Prime, invoices and ledger entries are automatically pushed to their Tally company with per-client company mapping.
  • Tally Prime XML API bridge (TDL-based)
  • Auto-push GST invoices on order confirm
  • Ledger entry creation per transaction
  • Per-client Tally company name mapping
  • Error handling & retry queue
  • Manual push option from admin panel
How per-client integrations work: Parsoncorp's admin panel includes a dedicated integrations page for each corporate client. For Google, enable Salesforce sync using Google's SF org + Tally sync using Google's Tally company. For Nvidia, only Tally. For Accenture, only Salesforce. Each client's integration is fully isolated — credentials encrypted per client. Built once, works for all current and future clients with zero code changes.
